# 使用模板与前置检查

## 前置检查

在调用本技能前，先确认：

| # | 检查项 | 说明 |
|---|---|---|
| 1 | 脚本已生成 | 建议先用 `tiktok爆款脚本制作skill` 生成完整脚本 |
| 2 | 脚本包含画面描述 | 每个阶段都要有 `画面` 描述 |
| 3 | Seedance 素材已准备 | 至少准备场景图、产品图、动作视频 |
| 4 | 钝刀镜头已补失败视频 | 只要脚本有钝刀失败切割，就必须准备失败动作视频 |
| 5 | 图参/视频参职责已区分 | 图参锁对象和场景，视频参锁动作、运镜、节奏、构图 |
| 6 | 时间线按现实时长切分 | 允许 0.5 秒粒度，不把 0.5-1 秒动作硬写成 3-4 秒 |

> 如果脚本只有口播，没有画面描述，本技能无法正常工作。

---

## Seedance 2.0 三条铁律提醒

调用 Seedance 2.0 时，默认遵守这 3 条铁律：

1. **角色锚点内嵌**：角色与风格描述必须写在每段正文开头，不单独列出
2. **始终使用 bare `@`**：所有参考对象在正文中必须带 `@`，禁止 `@图片N` / `@视频N`
3. **始终使用时间线**：每段正文必须包含连续的 `[Xs-Ys]` 时间线

额外规则：
- 不输出旧的编号映射格式
- 钝刀失败镜头缺少失败动作视频时，先提示补素材，不直接输出最终 Prompt
- 有图参时，正文优先改写成 `在@场景中`、`用@刀`、`展示@产品`
- 视频参考只写可观察到的动作、运镜、节奏、构图，不写成"@的钝刀失败节奏"这类语义标签
- 时间线必须贴合真实动作耗时，必要时使用 0.5 秒粒度

推荐素材清单写法：

```text
参考素材准备：
- 场景图：厨房场景
- 产品图：电动磨刀器
- 产品图：刀
- 产品图：磨好后的刀
- 动作视频：磨刀动作与节奏
- 动作视频：磨后切割验证
- 动作视频：失败切削动作与构图
```

## 常见问题

| 问题 | 原因 | 处理方式 |
|---|---|---|
| 正文没有角色描述 | 角色锚点写在外面了 | 把角色锚点直接融入每段正文开头 |
| 正文没有 @ 标记 | 漏掉了参考标记 | 所有需要参考的对象前面加 bare `@` |
| 正文没有时间线 | 漏掉了时间线标记 | 每段正文补齐连续的 `[Xs-Ys]` 时间线 |
| 单个动作被拖太长 | 把一个微动作当成整段来写 | 改成 0.5-1.5 秒子区间，或补上真实可见的后续动作 |
| 还残留 @图片N 语法 | 继续沿用了旧模板 | 改回 bare `@` 写法 |
| 钝刀镜头看起来不真实 | 没有失败动作视频 | 补独立失败动作视频 |
| 图参文案太重 | 把图里已有信息又写了一遍 | 改成 `在@场景中`、`用@刀` 这类最小必要表达 |
| 视频参写得太虚 | 把视频参考写成状态标签 | 改成 `按照@的切削动作和构图` 这种可观察表达 |
| Prompt 太长 | 一个片段塞了太多动作 | 拆分片段或去掉重复爽点 |
| POV 切割很怪 | 没写双手分工 | 明确谁固定对象、谁执行动作 |
| 画面广告感重 | 没有压制影棚语言 | 强化手机实拍感、自然光和生活化杂乱感 |
| 各段角色描述不一致 | 每段独立写导致用词漂移 | 确保每段角色锚点用词高度一致 |
