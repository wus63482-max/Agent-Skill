# 分镜 -> 视频提示词转化模板

## 通用转化公式

每个合并后片段都按以下结构生成：

```text
[前置锚点] + [时间线标记] + [动作序列（带 @）] + [环境/光影] + [音效]
```

生成前先判断：

1. 哪些阶段可以合并
2. 哪些对象必须被固定
3. 哪些动作必须挂 `@` 参考
4. 哪些结果必须压缩为更少的爽点
5. 角色锚点描述是否已内嵌到每段开头
6. 是否已经区分"图像参考锁对象/场景，视频参考锁动作/运镜/节奏/构图"
7. 单个动作是否按现实耗时切分，是否需要用 0.5 秒粒度

## Seedance 2.0 模板

### 标准输出

```text
### Script [N] AI 视频提示词

**Segment [N]（[总时间线位置]｜[合并的叙事阶段]）**:

**Prompt（Seedance 2.0）**:

视频场景，第一人称 POV 胸前固定运动相机视角，偏白男性手，黑色运动表，灰色连帽衫袖口。在@场景中，画面更亮更干净，iPhone 15 Pro 竖屏拍摄，手机原生画质。[0s-1.5s] 左手按住砧板上的番茄，右手用@刀按照@的切削动作和构图切出薄片。[1.5s-3.0s] 左手扶住砧板上的半颗洋葱，右手继续用@刀顺滑切开。[3.0s-3.5s] 最后双手把 @电动磨刀器摆在台面中央。环境音：清脆切番茄声、顺切洋葱声、产品轻放台面声。

**参考素材准备**：
- 场景图：厨房场景
- 产品图：电动磨刀器
- 产品图：磨好后的刀
- 动作视频：切割动作与构图

**Video Config**:
- Aspect Ratio: 9:16
- Duration: 6s
- Audio: Native
```

### 三条铁律检查

每段 Prompt 必须同时满足：

1. **角色锚点内嵌**：角色外貌/手部特征 + 场景描述 + 拍摄设备必须在正文开头
2. **bare `@` 参考**：所有需要参考素材的对象都带 `@`，禁止 `@图片N` / `@视频N`
3. **时间线标记**：正文必须包含连续的 `[Xs-Ys]` 时间线

额外约束：

- 参考图负责对象和场景锚定，不重复写"同一把""木质岛台""钝口主厨刀"这类已经被图片锁定的信息
- 参考视频只负责动作、运镜、节奏、构图，不要写成"@的钝刀失败节奏"这类语义过重的标签
- 时间线按现实时长分配，允许使用 0.5 秒粒度；不要把 0.5-1 秒动作拉长成 3-4 秒

### 钝刀镜头模板

只要是失败切割镜头，必须挂独立视频参考：

```text
视频场景，第一人称 POV 胸前固定运动相机视角，偏白男性手，黑色运动表，灰色连帽衫袖口。在@场景中，光线偏冷，iPhone 15 Pro 竖屏拍摄，手机原生画质。[0s-2.0s] 左手按住砧板上的番茄，右手用@刀按照@的切削动作和构图下压，把番茄切烂出汁。[2.0s-4.0s] 接着左手扶住砧板上的半颗洋葱，右手继续用@刀按照@的切削动作和构图下刀，刀路卡顿发涩。环境音：番茄被压碎的湿响、刀碰砧板的闷声、厨房底噪。

参考素材准备：
- 场景图：厨房场景
- 产品图：刀
- 动作视频：失败切削动作与构图
```

如果缺少"钝刀失败切割"视频，不要继续输出最终版 Seedance Prompt。

## 转化注意事项

1. 先锁场景和对象，再写动作
2. 每段正文开头必须内嵌角色锚点（不要单独输出全局角色段）
3. 一切切割动作都写明双手分工
4. 只要是切食材，必须明确写在砧板上切
5. 快剪段不要重复同类爽点
6. 所有需要参考的对象必须带 `@`
7. 每段正文必须包含连续的 `[Xs-Ys]` 时间线
8. 彻底清掉 `@图片N` / `@视频N` 旧编号标签
9. 钝刀镜头没有失败动作视频时，先补素材
10. 有图参时改写成 `在@场景中`、`用@刀` 这类最小必要名词
11. 视频参考只写成 `按照@的切削动作和构图`、`按照@的打磨动作和节奏`
12. 把短动作切成 0.5-1.5 秒子区间，不要用 3-4 秒硬撑一个完成极快的动作
